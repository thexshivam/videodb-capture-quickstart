# Security Policy

## Supported Versions

The following versions of the VideoDB Capture SDK are currently being supported with security updates:

| Version | Supported          |
| ------- | ------------------ |
| 1.x.x   | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

We take the security of our software seriously. If you have found a security vulnerability in the VideoDB Capture SDK or any of the examples in this repository, please report it to us as described below.

**Please do not report security vulnerabilities through public GitHub issues.**

### How to Report

Please send an email to [security@videodb.io](mailto:security@videodb.io) with "Security Vulnerability" in the subject line.

In your email, please include:

*   The version of the SDK/codebase where the vulnerability was found.
*   The type of vulnerability (e.g., XSS, buffer overflow, etc.).
*   Detailed steps to reproduce the vulnerability.
*   Any proof-of-concept code or screenshots.

### Response Timeline

We will acknowledge your report within 48 hours and provide an estimated timeline for a fix. We will notify you when the fix has been released.

Thank you for helping keep VideoDB secure!
